FPGA Baccarat

Abstract

A digital Baccarat engine implemented in SystemVerilog on the DE1-SoC. The design combines a simple datapath with a finite-state machine controller to deal cards, compute hand scores (mod 10), and display results on HEX displays and LEDs.

Goals

  • Design a clean datapath + controller architecture for a Baccarat game.
  • Drive HEX0–HEX5 with card values and LEDR with scores/win indicators.
  • Write self checking testbenches and perform post-synthesis checks.
  • Deploy to DE1-SoC and verify real time operation with push-button inputs.

System Design

The hardware is split into a datapath that stores cards and computes scores, and a controller FSM that sequences the game per Punto Banco rules. User presses KEY0 (slow_clock) to step through the game; KEY3 is a synchronous active-low reset.

Modules

  • Dealcard — Free-running counter on 50 MHz clocks through 1..13; sampled on user step to emulate randomness.
  • Scorehand — Maps face cards (10/J/Q/K) to 0, sums up to three cards, outputs (sum mod 10).
  • Statemachine — Sequencer for deal order and third-card rules.
  • Datapath — Wires registers, scoring, and display logic.


Datapath

Interfaces & I/O Mapping

  • Inputs: CLOCK_50 (dealcard), KEY0=slow_clock, KEY3=resetb (sync, active-low).
  • Displays: HEX0..HEX2 = Player cards, HEX3..HEX5 = Banker cards (active-low).
  • LEDs: LEDR[3:0] = Player score, LEDR[7:4] = Banker score, LEDR[8]=Player win, LEDR[9]=Banker win.

Results

  • Unit testbenches for scorehand, statemachine, and datapath pass (RTL and post-synthesis netlist).
  • On-board demo: correct dealing sequence, accurate scores (mod 10), and correct third-card behavior.
  • HEX accurately displays cards; LEDs indicate scores and winner per rules.
